# Useradmin and Superadmin Use Case

## Use Case Description

This is the process both Useradmin and Superadmin interact with the system related to Deka LLM products.

**Action and Response**

| Action by user | Response from system |
|---|---|
| Useradmin can view the metrics of usage activity API requests and total coins spent | System will display a dashboard of metrics based on coins spent |
| Useradmin can view the metrics of usage activity API requests and total tokens spent | System will display a dashboard of metrics based on tokens spent |
| Useradmin can configure coin balance alerting rules | System will save alert rules and applies them |
| Useradminâ€™s coin balance reaches threshold | System will send email notification within 5 minutes |
| Superadmin can view the metrics of usage activity API requests and total tokens | System will display the metrics |
| Superadmin can configure coin balance alerting rules | System will save alert rules and applies them |
| Superadmin can create vouchers for DekaLLM coin usage | System will create voucher code for LLM services only |
